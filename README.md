# amigo-native

[![CI](https://github.com/amigo-labs/amigo-native/actions/workflows/ci.yml/badge.svg)](https://github.com/amigo-labs/amigo-native/actions/workflows/ci.yml)
[![License: MIT](https://img.shields.io/badge/license-MIT-blue.svg)](https://github.com/amigo-labs/amigo-native/blob/main/LICENSE)

Rust-powered npm packages under the `@amigo-labs` scope.

Monorepo using [napi-rs](https://napi.rs) for native Node.js addons, cross-compile CI, and independent npm packages per crate.

## Packages

| Package | Description | vs JS original |
|:--------|:------------|:---------------|
| [`@amigo-labs/slugify`](./crates/slugify) | Unicode-aware slugification via `deunicode` | **3-7x faster** than `slugify` |
| [`@amigo-labs/argon2`](./crates/argon2) | Argon2id password hashing (sync + async) | **1.4x faster** than C-bindings, **2x faster** than WASM |
| [`@amigo-labs/xxhash`](./crates/xxhash) | XXH32/64/XXH3 hashing with batch + streaming API | **1.3-2.6x faster** at large data (>=1MB) |
| [`@amigo-labs/sanitize-html`](./crates/sanitize-html) | HTML sanitization via Mozilla's `ammonia` | **1.6-2x faster** than `sanitize-html` |
| [`@amigo-labs/csv`](./crates/csv) | CSV parsing/serialization via BurntSushi's `csv` | **1.4-3.5x faster** (via `parseToJson`) |

Full benchmark data: [BENCHMARKS.md](./BENCHMARKS.md) (auto-generated by CI).

## Quick start

```bash
pnpm install
pnpm build
pnpm test      # 45 tests
pnpm bench     # performance benchmarks
```

## Development

### Prerequisites

- Rust (edition 2024)
- Node.js >= 20
- pnpm

### Build

```bash
pnpm build          # all packages, release
pnpm build:debug    # all packages, debug (faster compile)
```

### Test

```bash
cargo test --workspace   # Rust unit tests
pnpm test                # Node.js tests (vitest)
```

### Benchmark

```bash
pnpm bench              # run all benchmarks (vitest bench)
pnpm bench:report       # run benchmarks + measure sizes + generate BENCHMARKS.md
```

### Lint

```bash
pnpm lint    # oxlint + cargo fmt --check + cargo clippy
```

## Adding a new package

```bash
./scripts/new-package.sh <package-name>
```

Then edit `crates/<name>/src/lib.rs` and `crates/<name>/Cargo.toml`.

## Architecture

```
amigo-native/
├── crates/
│   ├── slugify/          # @amigo-labs/slugify
│   ├── argon2/           # @amigo-labs/argon2
│   ├── xxhash/           # @amigo-labs/xxhash
│   ├── sanitize-html/    # @amigo-labs/sanitize-html
│   ├── csv/              # @amigo-labs/csv
│   └── _template/        # scaffold for new packages
├── scripts/
│   ├── new-package.sh    # package generator
│   ├── run-benchmarks.mjs
│   ├── measure-size.mjs
│   └── generate-report.mjs
├── .github/workflows/
│   ├── ci.yml            # lint + test + benchmark (on main)
│   └── release.yml       # cross-compile + npm publish
├── Cargo.toml            # workspace root
└── vitest.config.ts
```

Each crate is a standalone npm package with:
- Rust source in `src/lib.rs`
- napi-rs bindings (`#[napi]` macros)
- Platform-specific npm packages in `npm/` (6 targets)
- Tests in `__test__/` and benchmarks in `__bench__/`

## Release

Tag with `<crate-name>@<version>` (e.g. `slugify@0.1.0`) to trigger the release workflow. It cross-compiles for 6 targets and publishes to npm with provenance.

| Target | OS | Arch |
|:-------|:---|:-----|
| `x86_64-unknown-linux-gnu` | Linux | x64 |
| `x86_64-unknown-linux-musl` | Linux (Alpine) | x64 |
| `aarch64-unknown-linux-gnu` | Linux | ARM64 |
| `x86_64-apple-darwin` | macOS | x64 |
| `aarch64-apple-darwin` | macOS | ARM64 |
| `x86_64-pc-windows-msvc` | Windows | x64 |
