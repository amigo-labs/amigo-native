# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repo is

Monorepo of Rust-powered npm packages published under the `@amigo-labs` scope, using [napi-rs](https://napi.rs) for native Node.js addons. Each crate in `crates/` is a standalone npm package with its own benchmark, conformance, and release lifecycle.

**The portfolio thesis:** every shipped crate must be measurably faster than the best JS alternative on realistic median inputs. A crate that drifts below that bar gets deprecated — see the "Ported then deprecated" tag in `BACKLOG.md` and the post-mortems at `docs/post-mortems/`. Don't approach changes as pure code hygiene; if a refactor moves a crate between 🟢 Green / 🟡 Yellow / 🔴 Red per `docs/perf-review.md`, that's a portfolio-level change.

## Commands

```bash
# Build
pnpm build                                          # all crates, release
pnpm build:debug                                    # all crates, debug (faster compile)
pnpm --filter @amigo-labs/<name> build              # one crate

# Test
pnpm test                                           # all Node.js tests (vitest)
pnpm --filter @amigo-labs/<name> test               # unit tests for one crate
pnpm --filter @amigo-labs/<name> test:conformance   # upstream-parity tests for one crate
cargo test --workspace                              # Rust unit tests

# Single test / single bench
pnpm --filter @amigo-labs/<name> test -- -t "pattern"
pnpm --filter @amigo-labs/<name> bench -- -t "pattern"

# Benchmark + report
pnpm bench                                          # full suite (vitest bench)
node scripts/run-benchmarks.mjs --crates <name>     # one crate, writes bench-results-<name>.json at repo root
node scripts/run-benchmarks.mjs --only-changed      # crates whose source changed vs origin/main
node scripts/generate-report.mjs                    # rebuild docs/data.json from fresh shards
pnpm bench:report                                   # run-benchmarks + measure-size + parity-report + generate-report

# Lint / format
pnpm lint                                           # oxlint + cargo fmt --check + cargo clippy -D warnings
pnpm format                                         # prettier + cargo fmt + taplo format
```

Note: `vitest` is the single test runner for both unit tests (`__test__/`) and conformance tests (`__conformance__/`). `pnpm test` runs per-crate `test` scripts, `pnpm test:conformance` runs per-crate `test:conformance`.

## Architecture

### Per-crate layout

Every crate under `crates/*/` (except `_template`, `_ffi-bench`) follows:

```
crates/<name>/
├── src/lib.rs              # napi-rs bindings with #[napi] macros
├── Cargo.toml
├── package.json            # includes an "amigo" block — single source of truth
├── build.rs                # napi-rs code generation
├── index.js, index.d.ts    # generated by napi-rs
├── __test__/               # unit tests (vitest)
├── __conformance__/        # upstream-parity tests (vitest)
│   ├── parity.spec.ts      # our-vs-upstream on documented surface
│   ├── upstream.spec.ts    # run upstream's own tests against our binding
│   ├── fuzz.spec.ts        # fast-check property-based
│   ├── divergences.md      # documented gaps where full parity is impossible
│   └── README.md
├── __bench__/              # vitest bench suites, compares vs JS alternatives
├── npm/                    # 6 platform-stub npm packages (one per NAPI target)
├── MIGRATION.md            # only if not 100% drop-in
└── README.md
```

### Single source of truth: the `amigo` block

Each crate's `package.json` has an `"amigo": { title, description, tableDescription, replaces, vsJs, parity, status, competitors }` block. **Never** edit the README packages table or `docs/packages.json` by hand — run `node scripts/sync-registry.mjs` to regenerate both from the `amigo` blocks. CI enforces this.

### Benchmark → dashboard data flow

1. `pnpm bench` (or `scripts/run-benchmarks.mjs`) produces `bench-results-<name>.json` shards at repo root.
2. `scripts/generate-report.mjs` consolidates shards into `docs/data.json` and per-crate `docs/benchmarks/<name>.json`, and appends a row to the history.
3. `docs/index.html` + `docs/app.js` render `docs/data.json` on the [GitHub Pages dashboard](https://amigo-labs.github.io/amigo-native/).
4. CI (`ci.yml`) runs step 1 on changed crates per push to `main`. Force a full rerun with `[full-bench]` in the merge commit.

### Workspace root

- `Cargo.toml` declares the workspace and pins the **release profile** (`lto=true, codegen-units=1, strip=symbols, panic=abort`). Don't override these per-crate — they're portfolio-wide.
- `crates/_template/` and `crates/nanoid` are excluded from the Cargo workspace (`nanoid` is pure-JS after the Phase-C rescope, see `docs/perf-review.md`). `crates/_ffi-bench/` is the unpublished FFI-overhead baseline (see `docs/BASELINE.md` — 109 ns NAPI floor, 174 ns Buffer echo, etc.).

### BACKLOG ↔ perf-review lifecycle

- **`docs/perf-review/<name>.md`** is the one-page `rust-check` decision doc for every npm package we've evaluated — GO or NO-GO.
- **`BACKLOG.md`** is a flat "Don't Touch" list with tags (`[FFI]`, `[NATIVE]`, `[IO]`, `[PARITY]`, `[CALLBACK]`, `[SCOPE]`, `[JS-ENGINE]`, `[MEASURED]`, `[DEPRECATED]`) pointing to the NO-GO reviews. It's the "has this been considered?" reference; consult it before proposing any new port.
- **`docs/perf-review.md`** is the Nach-Sprint classification table for shipped crates (16 shipped, tagged Green/Yellow after Phase-C/D sprints).
- **`docs/post-mortems/`** holds post-mortems for Red-measured crates that got archived to `archived/`.

## Adding, changing, or killing a crate

- **New port:** `./scripts/new-package.sh <name>` scaffolds from `crates/_template/`. **Before writing code**, run the `rust-check` skill to produce a candidate review — a Green or Yellow prediction is required before a v0.1 spike gets scheduled. Red predictions live in `BACKLOG.md` with reasoning.
- **Audit a crate for convention drift:** use the `audit-crates` skill. It checks `__conformance__` layout (parity/upstream/fuzz + README + divergences), `test:conformance` + `test:all` scripts, 6-target `npm/` stubs, `docs/packages.json` registration, and marquee `PACKAGES` count.
- **Changes that affect speedup range or parity percentage:** update both `docs/perf-review.md` (verdict row) and the crate's `amigo` block (then run `sync-registry.mjs`). Output drift (e.g. hash-crate reseeding) is treated as a **breaking** change — bump major.
- **Killing a crate:** Phase-D path. `npm deprecate` → README warning → `MIGRATION.md` pointing to recommended JS alt → 3-month window → move to `archived/<name>/` + write `docs/post-mortems/<name>.md`. Precedents: `deep-equal`, `levenshtein`, `xml`.

## Conventions

- **Commits:** `feat(<crate>):`, `fix(<crate>):`, `perf(<crate>):`, `docs(<area>):`, `chore(...):`, `refactor(...):`.
- **Release:** tag `<crate>@<version>` (e.g. `slugify@0.2.0`) on `main` triggers cross-compile for 6 targets and `npm publish --provenance`. No manual publish.
- **Rust edition:** 2024. Node ≥ 20. Pnpm is the package manager.
- **FFI shape rules are binding.** Small-work-per-call (<µs in JS), hashmap-style lookups, `Vec<Object>`-over-FFI tree returns, and user-callback-over-FFI chains are all documented failure modes in `BACKLOG.md` / `docs/perf-review/xml.md` / `docs/perf-review/levenshtein.md`. Consult `docs/BASELINE.md` before proposing any new `#[napi]` signature.

## Skills available in this repo

- **`rust-check`** — single-package evaluation against the portfolio framework. Takes `<pkg>` arg, emits `docs/perf-review/<pkg>.md`, classifies GO / NO-GO.
- **`audit-crates`** — scans `crates/*/` for convention drift vs. the reference conventions (argon2/csv/sanitize-html/slugify/xxhash). Offers mechanical fixes interactively.
